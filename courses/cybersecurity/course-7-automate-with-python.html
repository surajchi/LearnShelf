Variables & Naming Conventions

📖DefinitionDEF-604

Variable: A named container that stores data in a computer's memory, whose value can change over time through reassignment.

PythonCopy

old_username = username

# Display the previous and current username

For Loops & While Loops

📖DefinitionDEF-606

Iterative statement: Code that repeatedly executes a set of instructions.

Loop variable: A variable controlling the iterations of a loop.

Python — For LoopCopy
computer_assets = ["laptop1", "desktop20", "smartphone03"]
for asset in computer_assets:
    print(asset)
What this does: Iterates through each element of the computer_assets list, printing each one in turn.
Syntax: The loop header (for asset in computer_assets:) ends with a colon; the body must be indented.
Python — range()Copy
for i in range(0, 5, 1):
    print(i)
What this does: range(start, stop, step) generates a sequence — here, 0 through 4. The start point is inclusive; the stop point is exclusive.
Shortcut: range(5) alone produces the identical sequence, since start defaults to 0 and step defaults to 1.
Python — While LoopCopy
login_attempts = 0
while login_attempts < 5:
    print("Login attempts:", login_attempts)
    login_attempts = login_attempts + 1
What this does: Continues looping as long as login_attempts < 5 evaluates to True, incrementing the counter each iteration until the condition becomes False.
Key difference from for loops: The loop variable (login_attempts) is assigned before the loop, not within the loop header itself.
⚠️WarningWRN-601

An infinite loop occurs when a while loop's condition never becomes False — for example, forgetting to increment the loop variable. If this happens, press Ctrl+C or Ctrl+Z to force-stop execution.

Loop Control: break & continue

Python — break vs. continueCopy
computer_assets = ["laptop1", "desktop20", "smartphone03"]

# break: exits the loop entirely
for asset in computer_assets:
    if asset == "desktop20":
        break
    print(asset)
# Output: laptop1

# continue: skips just this iteration
for asset in computer_assets:
    if asset == "desktop20":
        continue
    print(asset)
# Output: laptop1, smartphone03
Key difference: break stops the loop entirely the moment its condition is met — nothing after it in the sequence gets processed. continue only skips the current iteration, then resumes normally with the next item.

Scenario Challenge 1.1 — Loop Selection

You need to print every username in a list of exactly 10 known usernames. Separately, you need to keep prompting a user for a password until they enter the correct one. Which loop type fits each scenario, and why?

Answer: The first scenario (a known, fixed sequence) fits a for loop. The second scenario (unknown number of iterations, dependent on a condition) fits a while loop.

Interview Preparation

🎤Beginner QuestionINT-601

Q: When would you use a for loop instead of a while loop?

Ideal answer: "A for loop is best when you're iterating through a known sequence — a list, a string, or a range of numbers. A while loop is best when the number of iterations depends on a condition that isn't known in advance, like repeating an action until a user's login attempts are exhausted."

🎤Live Coding QuestionINT-602

Q: Write a loop that prints only the even numbers from 0 to 10.

Ideal answer: for i in range(0, 11, 2): print(i) — using the step parameter of range() to increment by 2 is more elegant than looping through every number and using a conditional to filter odds.

Practice Section

Correct — tuples cannot be changed after creation, unlike lists.
2. True or False: The condition in a for loop header determines how many times the loop runs.
False — a for loop runs once per item in its sequence; a while loop runs based on a condition.
3. What does the `continue` keyword do inside a loop?
Correct — continue skips just the current iteration, unlike break which exits entirely.

Defining & Calling Functions

📖DefinitionDEF-607

Function: A reusable block of code that performs a specific task, helping organize code and making it more readable and modular.

PythonCopy
def display_investigation_message():
    print("investigate activity")

# Calling the function
display_investigation_message()
What this does: The def keyword starts a function definition, naming it display_investigation_message. The indented line below is the function body. Calling the function (writing its name followed by parentheses) actually executes it.
Naming tip: Give functions names that describe what they do — it makes calling them later much easier to remember.
⚠️WarningWRN-602

Calling a function from within its own definition (recursion) without a condition that eventually stops it creates an infinite loop. For example, def func1(): func1() will call itself forever once invoked.

Parameters, Arguments & Return Statements

📖DefinitionDEF-608

Parameter: An object included in a function definition for use inside that function.

Argument: The actual data passed into a function when it's called.

Return statement: Uses the return keyword to send information back from a function to wherever it was called.

PythonCopy
def remaining_login_attempts(maximum_attempts, total_attempts):
    return maximum_attempts - total_attempts

remaining_attempts = remaining_login_attempts(3, 3)

if remaining_attempts <= 0:
    print("Your account is locked")
What this does: maximum_attempts and total_attempts are parameters. When called with (3, 3), those integers become the arguments — 3 passes into maximum_attempts, 3 passes into total_attempts. The function returns 0, stored in remaining_attempts, which then triggers the "locked" message.
Important: The return keyword is not a function — never place parentheses after it. Once Python executes a return statement, it immediately exits the function; any code after it inside the function never runs.

Global vs. Local Variables

Comparison — Global vs. Local Variables
TypeDefinedAccessible
Global variableOutside any functionEverywhere in the program, including inside functions
Local variableInside a function (including its parameters)Only within that function's body; deleted from memory once the function finishes running
Python — Variable ScopeCopy
username = "elarson"
print("1:" + username)

def greet():
    username = "bmoreno"
    print("2:" + username)

greet()
print("3:" + username)
Output: 1:elarson, then 2:bmoreno, then 3:elarson.
What's happening: Reusing the name username inside the function creates a separate local variable that shadows the global one only within the function — the global variable's value is never actually changed.
✅Best PracticeBP-601

Avoid combining global and local variables with the same name — it's confusing and error-prone. If you want a function to work with different values each time, use a parameter instead of relying on (and potentially reassigning) a global variable.

PEP 8, Comments & Readability

📖DefinitionDEF-610

Style guide: A manual providing guidelines for writing, formatting, and designing documents to promote clean, consistent output.

PEP 8 style guide: "Python Enhancement Proposal 8" — the standard stylistic guideline for Python code. Not mandatory, but ensures consistency and readability across the community.

Comment: A note programmers make about the intention behind their code, marked with # for single-line comments.

PEP 8's Core Recommendations
GuidelineSpecification
Line lengthKeep lines under 79 characters, including comments

# remaining_login_attempts() function takes two integer parameters,

Reviewing these four spots first when debugging a syntax error resolves the majority of cases quickly.

Scenario Challenge 2.1 — Global vs. Local Debugging

A colleague's function modifies a variable inside itself but complains the change isn't reflected outside the function. What's the most likely cause, and how would you explain it?

Answer guidance: The variable inside the function is almost certainly a local variable (even if it shares a name with a global variable) — Python creates a separate local copy rather than modifying the global one. String Indices & Bracket Notation

📖DefinitionDEF-611

Index: A number assigned to every element in a sequence indicating its position — indices start at 0 in Python.

Bracket notation: Indices placed in square brackets after a string or list to extract a specific element or slice.

Substring: A continuous sequence of characters within a larger string.

Indexing the String "h32rb17"
Characterh32rb17
Positive Index0123456
Negative Index-7-6-5-4-3-2-1
Python — SlicingCopy
device_id = "h32rb17"
print(device_id[0])
print(device_id[0:3])
Output: h, then h32.
Slicing rule: A slice's first index is inclusive; its second index is exclusive. [0:3] returns indices 0, 1, and 2 — stopping one position before index 3.
❗ImportantIMP-604

Strings are immutable in Python — once created, individual characters cannot be reassigned. Attempting device_id[0] = "x" raises an error. If you need to "change" a string, you must build and assign an entirely new one.

Python — Extracting a URL's Domain ExtensionCopy

Designing a Simple Algorithm

📖DefinitionDEF-612

Algorithm: A set of rules that solves a problem — steps that take input, perform tasks, and produce output.

🛰️SOC Example — Extracting IP Address PrefixesSOC-601

Problem: A security analyst needs the first three digits of multiple IP addresses to identify associated networks.

Algorithm breakdown: For one IP address, use string slicing to extract the first three characters. For multiple IP addresses stored in a list, wrap that slicing logic in a for loop and collect the results with .append().

    prefixes.append(ip[0:3])

What this does: Iterates through each IP address, slices out its first three characters, and appends the result to a new list — combining strings, lists, and loops into one small, reusable algorithm.

Regular Expressions: Symbols & Patterns

📖DefinitionDEF-613

Regular expression (regex): A sequence of characters forming a pattern, used to search for matches within strings — extremely useful for finding IP addresses, emails, or device IDs in log files without knowing the exact values in advance.

Python — Basic re.findall()Copy
import re

re.findall("ts", "tsnow, tshah, bmoreno")
Output: ['ts', 'ts'].
What's happening: re.findall(pattern, string) returns a list of every match. Without special symbols, the pattern is treated as a literal substring to find.
Regex Symbols for Character Types
SymbolMatches
\wAny alphanumeric character (letters, digits, and underscore)
\dAny single digit (0–9)
\sAny single whitespace character
.Any character at all, including symbols
\.A literal period character (backslash "escapes" its special meaning)
Regex Symbols to Quantify Occurrences
SymbolMeaning
+One or more occurrences of the preceding character
*Zero, one, or more occurrences of the preceding character
{n}Exactly n occurrences
{n,m}Between n and m occurrences (inclusive)
Python — Quantifiers in ActionCopy
import re

print(re.findall("\d+", "h32rb17"))
print(re.findall("\d{2}", "h32rb17 k825t0m c2994eh"))
print(re.findall("\d{1,3}", "h32rb17 k825t0m c2994eh"))
Output, line by line: ['32', '17'] — one or more consecutive digits. ['32', '17', '82', '29', '94'] — exactly two consecutive digits (a third digit in a row starts a new match). ['32', '17', '825', '0', '299', '4'] — between one and three consecutive digits.
Note: Python scans left to right; once a match completes, scanning resumes fresh from the next character, which is why three-in-a-row digit sequences split unexpectedly with {2}.

Building Complex Regex Patterns

🛰️SOC Example — Extracting Username:Attempts PairsSOC-602

Problem: Extract each username and login-attempt count from a string, ignoring employee IDs and department names.

Breaking it down: A username is \w+ (varying alphanumeric characters), followed by a literal colon :, a space \s, then the attempt count \d+ (varying digits).

Python — Combining Symbols into a PatternCopy
import re

pattern = "\w+:\s\d+"

employee_logins_string = "1001 bmoreno: 12 Marketing 1002 tshah: 7 Human Resources 1003 sgilmore: 5 Finance"

print(re.findall(pattern, employee_logins_string))
Output: ['bmoreno: 12', 'tshah: 7', 'sgilmore: 5'].
What's happening: Each regex component targets exactly one part of the pattern being searched for — the employee IDs and department names never match the pattern, so they're correctly excluded.
⚠️WarningWRN-603

Regex patterns can easily return too much or too little. Always test a regex pattern against representative sample data before relying on it for a real investigation — an overly broad pattern can silently miss the exact entries you actually need.

Opening & Reading Files: with, open(), read()

📖DefinitionDEF-614

File path: The location of a file or directory. An absolute file path starts from the root directory.

with: A keyword that handles errors and manages external resources, automatically closing a file once its indented block finishes.

open(): Opens a file, taking the file name/path as its first parameter and a mode ("r", "w", or "a") as its second.

Python — Reading a FileCopy
with open("update_log.txt", "r") as file:

print(updates)
What this does: Opens update_log.txt in read mode ("r"), reads its full content into the string variable updates via .read(), then prints it. The file automatically closes once the indented with block ends.

Parsing with split() and join()

📖DefinitionDEF-615

Parsing: The process of converting data into a more readable, usable format.

.split(): Converts a string into a list, separating it at a specified character (or any whitespace, if no argument is given).

.join(): Concatenates the elements of an iterable (like a list) back into a single string, using a specified separator.

Python — split() and join()Copy
approved_users = "elarson,bmoreno,tshah,sgilmore,eraab"
print("before .split():", approved_users)

approved_users = approved_users.split(",")
print("after .split():", approved_users)

approved_users = ",".join(approved_users)
print("after .join():", approved_users)
Output: A comma-separated string → a list of five usernames → back to the original comma-separated string.
Syntax difference: .split() is appended to the string being split. .join() is appended to the separator string, with the list to join passed in as its argument — the reverse pattern from most other methods you've learned.
Python — Reading, Parsing, then Rewriting a FileCopy
with open("update_log.txt", "r") as file:
    updates = file.read()
updates = updates.split()

# ... perform operations on the updates list ...

updates = " ".join(updates)
with open("update_log.txt", "w") as file:
    file.write(updates)
What this does: Reads a file into a string, splits it into a list for easier processing, then joins it back into a string before writing the (presumably modified) content back to the file.
Note the indentation: The .split() line isn't indented under the first with block — the file is deliberately closed as soon as its contents are captured, which is good practice.

Building a Login-Check Algorithm

🛰️SOC Example — Full Login-Check AlgorithmSOC-603

Goal: Given a log file of failed login usernames, flag any user with three or more failed attempts as locked out.

Python — Complete Login-Check FunctionCopy
def login_check(login_list, current_user):
    count = 0
    for username in login_list:
        if username == current_user:
            count = count + 1
    if count >= 3:
        print("Account locked")
    else:
        print("Login allowed")

with open("failed_logins.txt", "r") as file:
    usernames = file.read()
usernames = usernames.split()

login_check(usernames, "bmoreno")
What this does, step by step: (1) Reads and splits the log file into a list of usernames. (2) Defines login_check(), taking the full list and a specific username to check as parameters. (3) Loops through the list, incrementing a counter each time the target username appears. (4) Uses a final conditional to print the appropriate message.
This single example combines every major concept from all four modules of this course: files, loops, conditionals, functions, parameters, and string/list methods.

Debugger: A software tool that helps locate the source of an error and assess its causes, often using breakpoints — markers on specific lines that let you run and inspect code section by section, including checking variable values as they change.

Python — Debugging with Print StatementsCopy
new_users = ["sgilmore", "bmoreno"]
approved_users = ["bmoreno", "tshah", "elarson"]

def add_users():
    for user in new_users:
        print("line 5 - inside for loop")
        if user in approved_users:
            print("line 7 - inside if statement")
            print(user, "already in list")
        print("line 9 - before .append method")
        approved_users.append(user)

add_users()
print(approved_users)
What the print statements reveal: "line 9" prints twice, meaning .append() runs for both usernames — even the one already in the list. This isolates the bug: the append call should be inside an else block, not running unconditionally after the if statement.
The fix: Move approved_users.append(user) into an else clause so it only runs when the user isn't already present.
✅Best PracticeBP-603

Python reports errors one at a time, starting with the first it encounters. If your code has multiple errors, expect to fix them incrementally, rerunning after each fix — don't be discouraged if a new error appears after resolving the previous one; that's the normal debugging cycle, not a sign something is more broken than before.
