Chapter 04 · WinSCP

Moving files between your PC and a server — by drag and drop

SSH clients like PuTTY let you run commands on a remote machine. WinSCP solves the other half: copying files between your Windows desktop and a Linux server through a simple two-pane, drag-and-drop window — over the same secure connection.

01

Why do we need WinSCP?

SSH runs commands; WinSCP moves files

Where this comes from: The lecture introduces WinSCP as a "very useful tool." The scenario: "suppose this is your desktop" and you want to move files to a server. That's the job WinSCP does. from transcript
1 · The gap WinSCP fills

With PuTTY or the ssh command you can log into a server and type commands. But often you need to copy a file — a script, a config, a log — from your Windows desktop to the server, or bring one back. WinSCP gives you a friendly window to do exactly that.

2 · What it looks like

WinSCP shows two panels side by side: your local machine on one side, the remote server on the other. You move files simply by dragging them from one side to the other — no commands required.

02

Installing WinSCP

A few clicks — and skip the bundled extras

1 · Run the installer

Launch the WinSCP installer and keep clicking through the simple steps. When it offers extra add-ons you don't need, unselect them, then click Finish. WinSCP then opens.

Tip: Windows installers often bundle optional extras (toolbars, other apps). Untick anything you didn't ask for before finishing, so only WinSCP itself is installed. expanded
2 · First launch

The first time you open WinSCP you see its Login window — the screen where you'll tell it which server to connect to and how.

13 · Summary

Install is a quick Next → (unselect extras) → Finish. On first launch you land on the Login screen. First choice there: the transfer protocol.

03

Choosing a file-transfer protocol

SFTP, FTP, or SCP — different ways to carry the files

1 · The login screen

On the Login screen you can pick a protocol. The transcript lists the options it shows: SFTP, FTP, and SCP. Each is a different method of transferring files; you can transfer over whichever the server supports.

ProtocolWhat it isNote
SFTPSSH File Transfer ProtocolSecure; runs over SSH (port 22) — the common default
SCPSecure CopyAlso over SSH; simpler, older file copy
FTPFile Transfer ProtocolClassic file transfer; not encrypted on its own
Which to use? Because our server is reached over SSH on port 22, SFTP is the natural choice — it's secure and needs nothing extra beyond the SSH access we already set up. expanded
Memory trick
The S in SFTP and SCP = Secure (over SSH). Plain FTP has no S — no built-in security.
13 · Summary

WinSCP can move files over SFTP, FTP, or SCP. Over our SSH connection, SFTP is the easy secure default. Now we fill in the server details.

04

Connecting to the server

Host IP, port 22, username, password — and save it for next time

1 · Get the server's IP first

Just like with PuTTY, WinSCP needs the server's IP in the Host name field. So on the VM, read the IP: on CentOS use ifconfig; on Ubuntu use ip addr instead.

ifconfig (CentOS)
/
ip addr (Ubuntu)
Read the IP with the command that matches the distro
2 · Fill the login form

Back in WinSCP: put the IP in Host name, set Port number to 22 (same as before), enter User name = paul, and type the password.

3 · Save the session

You can Save these details, so next time you don't need to re-enter the IP. After saving, the session appears in the list on the left. Click Login.

4 · Accept the host key

On first login it asks to store the server's keys (the host key) — accept it, and provide the password again if prompted. You're now connected to the server's files.

Same connection as SSH: WinSCP over SFTP uses the exact SSH access you configured earlier (port 22, the SSH server running on the machine). If SSH logins work, WinSCP will too. expanded
13 · Summary

Enter the IP, port 22, username paul and password; save the session; accept the host key; log in. Now the real work — the two-pane transfer view.

The two-pane view — transferring files

Local on one side, remote on the other; drag to copy

1 · What you see after login

Once connected, WinSCP shows two file panels. One side is your local Windows machine; the other is the remote server. You browse folders on both, just like a normal file explorer.

2 · How you transfer

To move a file, simply drag it from one panel and drop it on the other. Drag left-to-right to upload to the server; drag right-to-left to download to your PC. That's the whole idea — no commands needed.

↔️
Real-life analogyTwo open drawers next to each other: your drawer and the server's drawer. Moving a file is just picking it up from one drawer and dropping it into the other.
13 · Summary

WinSCP's two panes let you drag files between your PC and the server to upload or download. To see it in action, we first make a file on the server.

06

Looking at & making files on the server

ls to see what's there, touch to create one

1 · Check what files exist

On the server side, list the current files with:

ls
Lists files in the current folder
2 · It's empty at first

In the lecture, ls shows no files yet — the home folder is empty. So we create one to have something to transfer.

3 · Create an empty file with touch

The touch command makes a new, empty file:

touch file
Creates an empty file named "file" (use any name you like)
What does touch do? If the file doesn't exist, touch creates it empty; if it already exists, it just updates its timestamp. It's the quickest way to make a placeholder file to test a transfer. expanded
4 · Now transfer it

Run ls again and the new file appears. In WinSCP's remote pane you'll now see it too — and you can drag it across to your desktop, or drag a desktop file over to the server the same way.

✗Expecting files to appear on the server without creating or uploading them.
✓Create with touch (or upload from your PC); confirm with ls.
13 · Summary

Use ls to inspect the server's files and touch to create one; then drag it between WinSCP's panes to move it. That's the full WinSCP workflow: connect, browse both sides, drag to transfer.
