No install needed — practise in the browser, free of cost

Tip: Search for a free browser-based Linux/terminal playground (for example a "JSLinux" or online-terminal service). Any real Linux terminal works — a VM, WSL, or a cloud shell — the commands below are identical everywhere. from transcript

Get any Linux terminal open (online, VM, or WSL). Now let's run the first handful of commands.

02

Your first commands

pwd, date, whoami, clear, ls

1 · Where am I? — pwd

pwd = Present Working Directory. It prints the folder you are currently in.

2 · Who am I? — whoami

whoami prints the currently logged-in user's name (e.g. paul). Notice the prompt also shows username@hostname and the current folder — so paul@host tells you the same thing at a glance.

3 · Date & time — date

date shows the system date and time. You can format it: date +%D for just the date, date +%T for just the time, or a custom format like date +%H:%M for hours:minutes.

4 · List files — ls

ls lists files and folders in the current directory. Add options for more detail: ls -lt (long list, sorted by time), ls -ltr (reverse — newest at the bottom), ls -lh (human-readable sizes like 76K).

Reading ls -l: the first character shows the type — a leading - means a normal file, d means a directory (folder). Colours in the terminal also distinguish them. from transcript
5 · Clean the screen — clear

clear wipes the terminal. Shortcut: press Ctrl + L — no command needed.

CommandDoes
pwdPrint present working directory
whoamiPrint current username
date +%TShow time (or +%D date, +%H:%M custom)
ls -ltrList, long, by time, reversed (newest last)
ls -lhList with human-readable sizes
clear (Ctrl+L)Clear the terminal
13 · Summary

These five orient you: where you are, who you are, the time, what's here, and a clean screen. Next, reading file contents.

03

Reading a file

cat, less, more

1 · Print a whole file — cat

cat file dumps the entire file straight onto the terminal. Good for short files.

2 · Read & search — less

less file opens the file in a scrollable viewer (an "editor-like" view, not printed to the terminal). Inside less:

KeyAction in less
/wordSearch top-to-bottom for "word"
?wordSearch bottom-to-top
nJump to next match
Shift+GGo to the last line
gGo to the first line
qQuit
3 · Page by page — more

more file shows a long file page by page. Press the down arrow / space to advance, q to quit. Unlike less, when you quit more the content stays printed on the terminal.

Memory trick
cat = dump it all. less = scroll & search (an editor view). more = page through and leave it printed.
13 · Summary

Use cat for short files, less to scroll and search big ones, more to page through. Next: editing.

04

Editing files

vi and nano

1 · Make a file — touch

touch newfile creates a new empty file. Check with ls. (Delete a file with rm newfile.)

2 · Editing with vi

vi newfile opens (and creates, if needed) the file in the vi editor. vi has modes:

Step in viKeys
Start typing (insert mode)press i
Stop typingpress Esc
Save & quit: then wq (i.e. :wq)
3 · Editing with nano

nano newfile is friendlier for beginners — it shows help hints at the bottom. Just type directly (no insert mode). To save & exit: Ctrl + X, then Y to confirm saving, then Enter to keep the same filename.

Tab trick
Type the start of a name and press Tab — the shell auto-completes it (e.g. new + Tab → newfile). Saves typing and typos.
vi vs nano: vi is powerful and everywhere but modal; nano is simpler with on-screen hints. Beginners often start with nano. from transcript
13 · Summary

vi: i to insert, Esc, :wq to save. nano: type, Ctrl+X, Y, Enter. Next: folders and moving around.

05

Working with folders & paths

mkdir, rm, cd, and absolute vs relative paths

1 · Make / remove a directory

mkdir newfolder = make directory. Remove an empty one with rmdir folder, or a folder with contents with rm -rf folder (recursive + force — be careful).

2 · Change directory — cd

cd newfolder = change directory (go in). cd .. goes up one folder; cd ../.. goes up two. You can chain: cd newfolder/newfolder to go several levels in at once. Plain cd (no argument) returns to your home folder.

3 · The root directory

/ is the root directory — the very top of the filesystem. cd / takes you there. Everything (all folders) lives under root. Each user also has a home folder at /home/<user>, shown by the ~ (tilde) sign.

4 · Absolute vs relative path

Absolute path starts from the root / — the full address, e.g. /home/paul/myfolder/newfolder. Relative path is written relative to where you currently are, e.g. just cd newfolder/newfolder. . means the current directory, .. means one level up.

Fig 5.1 — The folder hierarchy: an absolute path spells it out from / ; a relative path starts from where you are
Memory trick
Absolute = starts at / (always the same). Relative = starts at . (depends where you stand). .. = up, ~ = your home.
13 · Summary

mkdir/rmdir/rm -rf manage folders; cd moves around; paths are absolute (from /) or relative (from .). Next: copying and moving.

06

Copy, move & rename

cp and mv

1 · Copy a file — cp

cp <file> <destination>. Select the file, then give the destination path (relative or absolute). Example: cp mini.csv newfolder/.

2 · Copy a file's content into another file

cp source.csv backup.csv makes a same-content copy under a new name — a quick backup.

3 · Cut / move — mv

mv <file> <folder> moves (cut-paste) a file to another folder in one step: mv newfile newfolder. The original disappears from its old location.

4 · Rename — also mv

There's no dedicated rename command; mv does it. mv newfile newfile-modified renames the file.

TaskCommand
Copy file to foldercp mini.csv newfolder/
Copy into new file (backup)cp mini.csv mini-copy.csv
Move (cut-paste)mv newfile newfolder
Renamemv newfile newfile-modified
13 · Summary

cp copies, mv moves and renames. Next: slicing files with head, tail and sorting.

07

head, tail, sort & friends

head, tail, sort, uniq, split, wc, shuf, and the pipe |

1 · Top / bottom lines — head / tail

head -5 file shows the first 5 lines; tail -5 file shows the last 5. Handy for big data files.

2 · Sort — sort

sort file sorts lines A→Z. sort -r file reverses it (Z→A).

3 · The pipe | and uniq

The pipe | feeds one command's output into the next. To remove duplicates you first sort, then pass to uniq: sort file | uniq. (uniq only removes duplicates that are adjacent, which is why you sort first.)

4 · Split a file — split

split -l 4 file breaks a file into pieces of 4 lines each, creating files named xaa, xab, xac…

5 · Count lines — wc -l

wc -l file counts the number of lines — great for "how many records are in this data file?"

6 · Shuffle — shuf

shuf file randomly shuffles the order of lines.

CommandDoes
head -5 fileFirst 5 lines
tail -5 fileLast 5 lines
sort file / sort -r fileSort A→Z / Z→A
sort file | uniqRemove duplicate lines
split -l 4 fileSplit into 4-line files (xaa, xab…)
wc -l fileCount lines
shuf fileRandomly shuffle lines
13 · Summary

These slice, sort, dedupe, split, count and shuffle text — and the pipe | chains them together. Next: searching inside files.

08

Searching text — grep & egrep

Find a word (or several) inside files

1 · grep

grep "corey" test.csv searches a file for a word/pattern and prints (and highlights) matching lines. Much clearer than searching in less.

2 · egrep — search multiple words

egrep "corey|ana" test.csv finds lines matching corey OR ana — use the pipe inside quotes to add more terms.

grep "corey" test.csv
|
egrep "corey|ana" test.csv
Fig 8.1 — grep for one pattern; egrep for several at once
Very powerful: grep has many options for data searching. It's used constantly to filter output, e.g. history | grep sort. from transcript
13 · Summary

grep finds a pattern; egrep "a|b" finds several. Next: wildcards to match many filenames.

09

Wildcards

*, extensions, and {1..10} ranges

1 · Match by pattern — *

With 1000 files, * helps you target a subset. ls x* lists everything starting with "x". ls *.csv lists every file ending in .csv.

2 · Brace ranges — {1..10}

Curly braces expand a range. touch file{1..10} creates file1 through file10 in one command.

PatternMatches
x*Everything starting with x
*.csvEvery file ending in .csv
file{1..10}file1, file2, … file10
13 · Summary

Wildcards match many names at once: * for patterns, {1..10} for ranges. Next: comparing two files.

10

Comparing files

cmp and diff

1 · Are they identical? — cmp

cmp fileA fileB = compare. It just tells you whether two files differ and at which line — a yes/no style answer.

2 · What's different? — diff

diff fileA fileB shows the actual differences, line by line. Lines only in A are marked with -, lines only in B with +. diff -u gives a clearer unified view. It's line-by-line, not word-by-word.

13 · Summary

cmp = same or not; diff = exactly what changed. Next: finding files on disk.

11

Finding files

find and locate (with updatedb)

1 · find — search live

find <path> -name <name> searches a location (and everything under it) for a file. Example: find ./ -name test.csv searches the current folder tree. It is case-sensitive (capital T ≠ small t), and supports wildcards: find ./ -name "*.csv".

2 · locate — search a database

locate test.csv is very fast because it searches a prebuilt database (cache) rather than the live disk. Downside: new files won't appear until the database is refreshed.

3 · Refresh the database — updatedb

Before relying on locate for recent files, run sudo updatedb (needs root/sudo). Then locate will find them.

find vs locate: find is always current but slower (walks the disk); locate is instant but only as fresh as the last updatedb. from transcript
13 · Summary

find for live, case-sensitive searches; locate (+ updatedb) for instant lookups. Next: handy utility commands.

12

Utility commands

bc, cal, uptime, script, history, --help, man, which

1 · Calculator — bc

bc is a command-line calculator. Type it, then do math like 8 * 9 or 9 - 8. Exit with Ctrl + C.

2 · Calendar — cal

cal shows the current month. cal 2020 shows a whole year; cal 1 2022 shows a specific month (January 2022).

3 · Server uptime — uptime

uptime shows how long the machine has been running, how many users are logged in, and the load average.

4 · Record your terminal — script

script records everything you type and its output into a file (default typescript). Stop with exit, then read it with cat typescript. Great for capturing a session you're learning or for evidence of a task.

5 · Command history — history

history lists commands you've run before. Combine with grep: history | grep sort. (The up-arrow also recalls the previous command.)

6 · Getting help — --help, man, which

ls --help shows a command's syntax and options (pipe to more if long: ls --help | more). man ls opens the full manual. which ls shows which executable runs (e.g. /usr/bin/ls).

CommandDoes
bcCalculator (Ctrl+C to exit)
cal / cal 2020 / cal 1 2022Calendar: month / year / specific month
uptimeHow long the server has run + load
scriptRecord terminal activity to a file
historyList previously used commands
ls --help / man ls / which lsOptions / manual / executable path
13 · Summary

A grab-bag of everyday helpers — calculator, calendar, uptime, session recording, history, and the three ways to get help. Next: aliases.

13

Alias — shortcuts for long commands

Type less by naming your own commands

1 · Create an alias

alias l="ls -ltr" makes l a shortcut for ls -ltr. Now typing l runs the long command. Great for long or repeated command combinations.

2 · List aliases

alias -p lists the aliases currently defined.

Note on the temporary nature: an alias made this way is temporary — it vanishes when you log out. To make it permanent, add it to your ~/.bashrc file and source it (see the environment-variables chapter for the same idea). from transcript
13 · Summary

alias short="long command" saves typing; make it permanent via ~/.bashrc. Next: compressing files.

14

Compress & decompress

gzip, gunzip, tar, zip, unzip

1 · One file — gzip / gunzip

gzip -k test.csv makes a compressed test.csv.gz (the -k keeps the original). gunzip test.csv.gz decompresses it back.

2 · A folder — tar

gzip won't compress a folder. Use tar (tape archive): tar -czf myfolder.tar.gz newfolder compresses a whole folder into one file. The flags: c=create, z=gzip, f=file. Decompress with tar -xzf myfolder.tar.gz (x=extract).

3 · Multiple files into one zip — zip / unzip

zip myfiles.zip fileA fileB bundles selected files into a .zip. unzip myfiles.zip extracts them. List a zip's contents without extracting using unzip -l myfiles.zip.

TaskCommand
Compress a file (keep original)gzip -k test.csv
Decompressgunzip test.csv.gz
Compress a foldertar -czf out.tar.gz folder
Extract a foldertar -xzf out.tar.gz
Zip chosen fileszip out.zip fileA fileB
Unzip / listunzip out.zip / unzip -l out.zip
13 · Summary

gzip for one file, tar for folders, zip for bundling several — each with its inverse. Next: downloading from the internet.

15

Downloading files & calling APIs

wget and curl

1 · Download — wget

Copy a file's download link, then wget <url> saves it to the current folder. Example: wget https://www.python.org/…/Python-3.11.tgz.

2 · API calls & downloads — curl

curl can also download and, importantly, make API calls (HTTP GET/POST). Example: curl http://numbersapi.com/random returns a random fact printed to the terminal; run it again for a new response.

wget vs curl: wget is simplest for grabbing files; curl is a Swiss-army knife for HTTP — GET, POST, headers, APIs. from transcript
13 · Summary

wget url downloads; curl downloads and talks to APIs. Next: installing software.

16

Package management

yum, dnf, apt, rpm

1 · The right tool depends on the distro

Ubuntu / Debian use apt. Red Hat / CentOS / Fedora use yum (or newer dnf). Corporate servers are often Red Hat-based, so the lecture focuses on yum.

2 · Install — needs privileges

yum install nginx fails as a normal user: "This command has to be run with superuser privileges." Prefix with sudo: sudo yum install nginx, enter your password, and confirm with y.

Why sudo? Installing software changes the whole system, so it needs admin rights. Whether you have sudo is decided by the admin — if you can't use it, ask them. from transcript
3 · Check & list packages — rpm, yum list

rpm -qa lists all installed packages (grep to filter: rpm -qa | grep java). yum list installed | grep nginx checks if something is installed. yum list available | grep mysql shows what versions you can install.

TaskCommand
Install (Red Hat family)sudo yum install nginx
Install (Debian/Ubuntu)sudo apt install nginx
List installedrpm -qa (or yum list installed)
Is X installed?rpm -qa | grep X
What versions available?yum list available | grep X
13 · Summary

Package tool follows the distro: apt (Debian) or yum/dnf (Red Hat); installs need sudo; rpm -qa and yum list inspect packages. Next: controlling services.

17

Service control — systemctl

start, stop, status, and list services

1 · What's a service

Things like nginx, httpd (Apache), firewalld, or ssh run as services. You control them with systemctl.

2 · status / start / stop

systemctl status firewalld.service shows if it's active (running). A freshly installed service is inactive until started. systemctl start nginx starts it (asks for root/sudo), systemctl status nginx confirms "active (running)", and systemctl stop nginx stops it (status becomes inactive/dead).

3 · List all services

systemctl list-units --type=service --all lists every service and its state — like a Windows "Services" manager.

TaskCommand
Statussystemctl status nginx
Startsystemctl start nginx
Stopsystemctl stop nginx
List all servicessystemctl list-units --type=service --all
13 · Summary

systemctl start/stop/status control any service; list-units shows them all. Next: environment variables.

18

Environment variables

printenv, export, ~/.bashrc, source

1 · What they are

An environment variable is a value defined at the OS level so programs can find things — for example JAVA_HOME tells the system where Java lives. PATH lists folders the OS searches for executables.

2 · See them — printenv

printenv lists all environment variables (SHELL, HOSTNAME, PWD, PATH, and more).

3 · Set one (temporary) — export

export JAVA_HOME=/usr/lib/jvm/java-17 sets it, and export PATH=$JAVA_HOME/bin:$PATH puts its binaries on the path. Check with printenv. Example use: switch which Java version runs (java -version will then show 17 instead of 11). These changes vanish on logout.

4 · Make it permanent — ~/.bashrc + source

Add the variable to your ~/.bashrc (a hidden per-user file; see hidden files with ls -a), export it there, then apply without logging out via source ~/.bashrc. Now it persists across logins. This is user-specific — other users won't see it.

export VAR=value

→

13 · Summary

printenv reads, export sets (temporarily); ~/.bashrc + source makes it permanent per user. Next: string/text operations.

19

String & text operations

awk, cut, sed, tr, fold

1 · Print a CSV column — awk

awk -F, '{print $2}' test.csv prints the 2nd column. -F, sets the field separator to a comma (CSV). $NF is the last column; {print $1,$2} prints several.

2 · Slice fixed characters — cut

cut -c 1-2 file prints characters 1–2 of every line (e.g. the state prefix of vehicle numbers). Change the range for different slices.

3 · Print / replace specific lines — sed

sed -n '5p' file prints only line 5. Replace a word: sed -n 's/business/finance/g' file (s = substitute, g = global). Note: by default this shows the change on the terminal, it doesn't rewrite the file unless you tell it to.

4 · Translate case / delete chars — tr

tr = translate. tr a-z A-Z < file converts lower to upper case (or reverse). tr -d '%' < file deletes a character. Input comes from a file via <.

5 · Vertical output — fold

echo abcde | fold -w1 prints each character on its own line (folds at width 1).

CommandDoes
awk -F, '{print $2}' filePrint 2nd CSV column ($NF = last)
cut -c 1-2 fileCharacters 1–2 of each line
sed -n '5p' filePrint line 5
sed 's/business/finance/g' fileReplace word globally
tr a-z A-Z < fileLower → upper case
fold -w1One character per line
13 · Summary

awk for columns, cut for character slices, sed for line print/replace, tr for case/deletes, fold for vertical output. Next: users and privileges.

20

Switching users & sudo

su, sudo, exit

1 · Become another user — su

su - (or su) switches to the root user; it asks for root's password. You need to know the target user's password to switch to them.

2 · Go back — exit

exit returns you to your previous user (e.g. root → back to paul).

3 · Run one admin command — sudo

If you're not root, prefix an admin command with sudo (e.g. sudo yum install…, or stopping a service). Whether you may use sudo is granted by the admin — if it's denied, ask them.

13 · Summary

su switches user, exit returns, sudo runs a single admin command. Next: file permissions.

21

File permissions & ownership

Reading rwx, chmod, chown, chgrp

1 · Why it matters

cat sensitive.file can fail with Permission denied. Understanding permissions explains why, and how to fix it.

2 · Reading the permission string

ls -ltr shows something like -rwxr-x r--. After the type character, permissions come in three groups of three: owner, group, others. Each group is r (read), w (write/edit), x (execute). A - means that permission is missing. The listing also shows the file's owner and group owner (e.g. root root).

- (type)
|
rwx (owner)
|
r-x (group)
|
r-- (others)
Fig 21.1 — Permission string: owner / group / others, each as read-write-execute
3 · Change permissions — chmod

chmod = change mode. Syntax: chmod <who><+/-><perm> file. Who: u=user/owner, g=group, o=others, a=all. + adds, - removes. Example: chmod o+r sensitive.file lets others read it. Only the file's owner (or root) can change its permissions.

4 · Change owner / group — chown / chgrp

chown paul rootfile changes the file's owner to paul. chgrp paul rootfile changes its group owner. (Both typically need root.)

TaskCommand
Let others readchmod o+r file
Give owner executechmod u+x file
Give everyone readchmod a+r file
Change ownerchown paul file
Change group ownerchgrp paul file
13 · Summary

Permissions are owner/group/others × rwx; chmod edits them, chown/chgrp change ownership. A "Permission denied" now makes sense. Next: system, memory and processes.

22

System, memory & process commands

free, top, df, du, uname, lscpu, lsblk, ps — plus networking, users & automation

1 · Memory — free

free -h shows RAM: total, used, free (human-readable), plus swap. free -th adds a Total row.

2 · Live monitor — top

top is a live monitor of CPU and memory, showing which process consumes the most. Quit with q.

3 · Disk usage — du / df

du -h myfolder shows how much space a folder (and its subfolders) uses. df -h shows each filesystem/partition: size, used, available, and use% — high use% (80-99%) is a red flag.

4 · System info — hostname, lscpu, arch, lsblk, uname

hostname = machine name. lscpu = CPU cores/threads/architecture. arch = 32- or 64-bit. lsblk = storage devices & partitions (root, swap, home). uname -a or cat /etc/os-release = OS name/version (e.g. Red Hat Enterprise).

5 · Processes — ps

A process is any running application, script, or task, each with a PID (process ID). ps / ps -ef lists processes and their PIDs. When you start a service (e.g. nginx), it gets a PID used to track it. You can then inspect or (with kill <PID>) stop it.
