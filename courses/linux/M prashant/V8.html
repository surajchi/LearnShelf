A – D

awk, alias, apt, at · bg, bc, bash · chmod, chown, chgrp, cp, cal · df, du, diff, date

How to use this chapter

Each letter gets a card. The coloured tag on the right of each command tells you which task group it belongs to — files network process user/perms — so you're revising the grouping and the alphabet at the same time.

A

A — text processing, shortcuts & packages

awkVery helpful for text processing: print a chosen column, use a field separator, and filter data. filesawk -F, '{print $2}' data.csv
aliasCreate a shortcut for a longer command. shellalias ll="ls -ltr"
aptPackage management — install a package. Used especially on Ubuntu. appssudo apt install nginx
atSchedule a job to run once at a given time. processecho "./backup.sh" | at 02:00
at vs crontab: at schedules a one-time job at a specific moment; crontab schedules a repeating job on a recurring pattern. added for clarity
B

B — background jobs, maths & the shell itself

bgResume a stopped job in the background. processCtrl+Z to pause, then: bg
bcThe basic calculator — do arithmetic right in the terminal. utilityecho "5*3" | bc
bashThe Bourne Again Shell — used to run your scripts. shellbash myscript.sh
C

C — permissions, ownership & copying

chmodChange mode — modify a file's permissions. permschmod +x script.sh · chmod 755 file
chownChange the file's ownership (which user owns it). permssudo chown paul file.txt
chgrpChange the group ownership of a file. permssudo chgrp developers file.txt
cpThe simplest one — copy a file. filescp -r folder/ backup/
calShow a calendar — useful to check dates for a specific month or year. utilitycal 2026 · cal 7 2026
D

D — disk space, differences & the clock

dfDisk free — check the filesystem's disk space. systemdf -h
duDisk utilisation — the space used by one folder or file. systemdu -sh /var/log
diffFile comparison — show the differences between two files. filesdiff -u old.conf new.conf
dateThe system date and time. utilitydate · date +%T
df -h
→
"disk is 96% full"
→
du -sh /*
→
"/var/log is 40G"
Fig 1.1 — The classic pairing: df tells you the disk is full, du tells you what filled it.
Memory trick
df = disk free (whole disk) · du = disk usage (one folder). ch* commands all change something: mode, owner, grp.
02

E – H

env, export, echo · find, file, fg, free · grep, gzip, groupadd, groupdel · history, head, hostname

E

E — environment variables & printing

envCheck the environment variables currently set. envenv | grep JAVA
exportSet an environment variable. envexport JAVA_HOME=/usr/lib/jvm/java-17
echoSimple: print anything to the terminal, or redirect it into a file. consoleecho "hi" · echo "hi" >> notes.txt
F

F — finding, foreground & free memory

findSearch for a file or directory across a whole directory hierarchy. filesfind /var -name "*.log" -size +10G
fileGet information about a file's type. filesfile report.pdf → PDF document
fgForeground — resume a job in the foreground. Process management. processfg %1
freeCheck the status of free RAM and available RAM. systemfree -h
G

G — the famous one, plus compression & groups

grepSo popular — used for pattern searching inside files. filesgrep -rin "error" /var/log/
gzipCompress a file. filesgzip bigfile.csv · gunzip bigfile.csv.gz
groupaddCreate a group — part of user management. userssudo groupadd developers
groupdelDelete a group. userssudo groupdel developers
H

H — history, the top of a file & the machine's name

historyHeavily used — check which commands you recently executed. shellhistory | grep ssh
headSee the top-most specific lines of a file. fileshead -n 5 app.log
hostnameCheck a server's hostname. networkhostname · hostname -I
Memory trick
head = top of the file · tail = bottom (chapter 5). export pushes a variable out to child programs; env shows what's already out there.
03

I – L

ifconfig, ip, id, iptables · jobs · kill · ls, less, ln, locate

I

I — IP addresses, identity & the firewall

ifconfigHeavily used — check your server's IP. networkifconfig
ipThe modern one — network interface and IP-related information. networkip addr · ip route
idInformation about a user: their groups, UID and GID. usersid riya
iptablesManage firewall rules. networksudo iptables -L
J

J — the only real answer

jobsList down the current jobs in this shell and their state — stopped, or still active. processjobs
Interview tip: J is the letter people freeze on. jobs is the answer — and it pairs naturally with bg and fg, which is a good thing to say out loud. added for clarity
K

K — ending a process

killHeavily used — kill or terminate a process by its PID. processkill 4821 · kill -9 4821 (force)
L

L — listing, reading, linking & locating

lsVery popular — list a directory's contents. filesls -ltr
lessSimple — what we use for reading bigger files (with forward & reverse search). filesless /var/log/syslog
lnCreate hard and soft links. filesln -s /opt/app/v2 /opt/app/current
locateA substitute for find — search for files, a slightly easier option. fileslocate nginx.conf

04

M – P

mkdir, mv, more · netstat, nano, nice · openssl · ps, printenv, pwd, passwd

M

M — making, moving & paging

mkdirMake directory — create a folder. filesmkdir -p project/logs
mvMove a file from one directory to another (like cut-paste) — or rename it. filesmv old.txt new.txt
moreShow a command's output, or read a file, page by page. filesmore app.log · ps -ef | more
N

N — networking, an easier editor & priority

netstatLots of network information: protocols, network interfaces, ports and routing tables. Very helpful. networknetstat -tulpn
nanoA very good substitute for the vi and vim editors — friendlier for beginners. filesnano config.yml
niceThe nice value — sets a process's priority (process management). processnice -n 10 ./heavy.sh
O

O — rarely used, but have an answer ready

opensslA toolkit for TLS/SSL protocols — certificates, keys, encryption. Not used much day to day, but it's a solid answer for the letter O. networkopenssl s_client -connect example.com:443
P

P — processes, paths & passwords

psCheck process status — what's running and its PID. processps -ef | grep nginx
printenvKind of the same as env — print the environment variables. envprintenv PATH
pwdCheck your location — you get the full path. filespwd
passwdChange a password. userspasswd · sudo passwd riya
Memory trick
P is the richest letter: process status, print env, present directory, password. If you blank in an interview, "P" always has an answer.
05

Q – T

quota · rsync, rm, rmdir, reboot · sudo, sed, sort, systemctl · top, tail, tar, telnet, tee

Q

Q — the hardest letter

quotaQ has very few options. quota displays the disk usage and limits for a user or group. That's the one to name. usersquota -u paul
R

R — copying, removing & restarting

rsyncA fast and versatile file copying tool — great for syncing folders and backups. networkrsync -avz /data/ user@host:/backup/
rmSimple — delete a file. filesrm file.txt · rm -r folder/
rmdirRemove a directory (an empty one). filesrmdir emptyfolder
rebootReboot the system. systemsudo reboot
S

S — privilege, streams & services

sudoHeavily used — perform an action with super-user privileges as a normal user (install a package, start/stop a service). userssudo apt install nginx
sedStream editor — find and replace text. filessed 's/error/ERROR/g' app.log
sortSort data alphabetically (or numerically). filessort names.txt · sort -r names.txt
systemctlCheck a service's status, and start, stop or restart it. servicessudo systemctl restart nginx
T

T — the busiest letter

topHeavily used — check processes and CPU utilisation percentage-wise. It's a whole monitoring tool in one command. processtop (press q to quit)
tailSee some bottom lines of a file. With -f it follows a log live. filestail -f /var/log/syslog
tarTape archive — archive and compress a whole directory. filestar -czf backup.tar.gz folder/
telnetCheck connectivity based on IP and port — is that specific port open? networktelnet 10.0.0.5 3306
teeWrite output to a file and print it on the terminal at the same time. Not used a lot, but sometimes exactly right. console./deploy.sh | tee deploy.log
bash — why tee exists
$ ./deploy.sh > deploy.log        # saved, but you see nothing
$ ./deploy.sh | tee deploy.log    # saved AND printed live
Deploying... OK
Memory trick
tee is named after a T-pipe in plumbing: one stream in, two streams out — one to the file, one to your screen.
06

U – Z

uniq, useradd, usermod, umount, umask · vi, vim · wget, wc, whoami, which · xxd · yum, yes · zip

U

U — unique values, users & default permissions

uniqGoes with sort — when you want only the unique values from a file. filessort names.txt | uniq
useraddCreate a user — user management. userssudo useradd -m riya
usermodModify an existing user (e.g. add them to a group). userssudo usermod -aG sudo riya
umountUnmount a mounted filesystem or disk. systemsudo umount /mnt/data
umaskSet the default permissions for newly created files and directories. permsumask 022
Note on usermod -aG: always include the a (append). Plain -G replaces all of the user's secondary groups, silently dropping the rest. common mistake
V

V — the editors

viHeavily used for file editing. filesvi file → i to insert, Esc, :wq to save
vimVi improved — same idea, more features (syntax colours, etc.). filesvim config.yml
W

W — downloading, counting & identity

wgetDownload files from the internet (also FTP and more). networkwget https://site.com/app.tar.gz
wcWord count — how many lines, words and characters. fileswc -l data.csv
whoamiWho is the current user? userswhoami
whichFor any command, find out where it is being sourced from (its binary path). appswhich python3 → /usr/bin/python3
X

X — the trick letter

xxdCreates a hex dump of a given file or of standard input. Not very useful day to day, but at least it's a real answer for X. utilityxxd file.bin | head
Y

Y — packages & endless output

yumPackage management, based on Red Hat / CentOS. appssudo yum install nginx
yesRun a task repetitively — e.g. keep a script looping when it has no while loop of its own, or auto-answer "y" to prompts. utilityyes | sudo apt install nginx
Z

Z — the last one

zipCompress and archive your files so they transfer easily. fileszip -r out.zip folder/ · unzip out.zip

10 Key Points

  • 1The A–Z question isn't about trivia — it's a fast check that you've actually used Linux.
  • 2The hard letters are J, O, Q, X, Y. Memorise one each: jobs, openssl, quota, xxd, yes.
  • 3T is the richest letter: top, tail, tar, telnet, tee.
  • 4df = whole filesystem; du = one folder. Interviewers love this pair.
  • 5find walks the live disk; locate reads a fast database (needs updatedb).
  • 6bg, fg, jobs, nice, kill, ps, top are all one family: process management.
  • 7chmod/chown/chgrp/umask are all one family: permissions.
  • 8yum = Red Hat/CentOS, apt = Ubuntu. Naming both shows distro awareness.
  • 9tee = write to a file and the screen at once; > only writes to the file.
  • 10Grouping commands by task (chapter 07) is how you recall them; the alphabet is just the drill.

5 Interview Questions

Name a Linux command starting with J, O, Q and X.
These are the four that trip people up. J → jobs (list current background jobs, stopped or active). O → openssl (a toolkit for TLS/SSL). Q → quota (display disk usage and limits for a user or group). X → xxd (create a hex dump of a file or standard input).
What is the difference between df and du?
df (disk free) reports space at the filesystem/partition level — total, used, available and use%. du (disk utilisation) reports the space consumed by one particular directory or file. The workflow is: df -h tells you the disk is full, then du -sh /* tells you which folder filled it.
What is tee and why would you use it?
tee writes output to a file and the terminal at the same time. With a plain redirect (cmd > file) the output is saved but invisible while it runs. cmd | tee file lets you watch a long deploy or build live and keep the log. It's named after a T-shaped pipe: one input, two outputs.
Difference between find and locate?
Both search for files. locate is a faster, easier substitute because it queries a pre-built database — but that database is only as fresh as the last updatedb, so brand-new files can be missing. find walks the live filesystem, so it's always accurate and supports rich conditions (-name, -mtime, -size).
Which commands belong to process management?
ps (process status & PID), top (live CPU/memory monitor), kill (terminate by PID, -9 to force), jobs (list jobs in this shell), bg (resume in background), fg (resume in foreground), and nice (set priority). Grouping them like this is exactly what an interviewer wants to hear.

Common Mistakes

✗"usermod -G sudo riya adds her to sudo."
✓Plain -G replaces all secondary groups; use -aG to append.
✗"locate always finds every file."
✓It reads a database — a new file may be missing until updatedb runs.
✗"gzip can compress a folder."
✓gzip is single-file only; use tar -czf or zip -r for folders.
✗"rmdir deletes any directory."
✓rmdir only removes empty directories; use rm -r otherwise.
