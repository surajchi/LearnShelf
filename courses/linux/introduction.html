Linux Introduction

Linux is a free, open-source operating system kernel first released by Linus Torvalds in 1991. Paired with system tools it becomes a full operating system — a distribution — such as Ubuntu, Debian, Fedora, or Arch. It runs most of the world's servers, Android phones, routers, and supercomputers.

Why learn Linux?

  • It powers the servers you will deploy code to.
  • The command line is fast, scriptable, and repeatable.
  • Skills transfer across every distribution and cloud provider.

The shell

You talk to Linux through a shell — a program that reads commands and runs them. The most common shell is bash. When you open a terminal you see a prompt, waiting for input: 

user@machine:~$

Type a command and press Enter. Try these first three:

whoami     # print your username
pwd        # print working directory (where you are)
ls         # list files in the current directory

Anatomy of a command

Most commands follow the same shape:

command  -options  arguments

For example:

ls -l /home
  • ls — the command (list).
  • -l — an option (long, detailed format).
  • /home — an argument (which directory to list).

Getting help

Every command documents itself. Use man (manual) or --help: 

man ls
ls --help

Summary

  • Linux is an open-source OS running most servers and devices.
  • You control it through a shell such as bash.
  • Commands take the form: command -options arguments.
  • Use man or --help when stuck.
