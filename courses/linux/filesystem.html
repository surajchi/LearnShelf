The Linux Filesystem

In Linux, everything is a file — documents, devices, even directories. All files live in a single tree that starts at the root, written as /. There are no drive letters like C:; every disk is mounted somewhere inside that one tree.

Key directories

  • / — the root of everything.
  • /home — user home folders (/home/alice).
  • /etc — system configuration files.
  • /bin, /usr/bin — installed programs.
  • /var — logs and changing data.
  • /tmp — temporary files, cleared on reboot.

Where am I?

Your current directory is where the shell is "standing". Print it with: 

pwd        # e.g. /home/alice

Moving around

Use cd (change directory):

cd /etc        # go to an absolute path
cd Documents   # go into a subfolder (relative)
cd ..          # go up one level
cd ~           # go to your home folder
cd -           # go back to the previous folder

Absolute vs relative paths

  • Absolute paths start at root: /home/alice/notes.txt. Same meaning everywhere.
  • Relative paths start from where you are: notes.txt or ../photos.

Listing and inspecting

ls          # list files
ls -a       # include hidden files (start with .)
ls -l       # long format: permissions, owner, size, date
ls -lh      # sizes in human-readable units (KB, MB)

Creating and removing

mkdir projects        # make a directory
touch notes.txt       # make an empty file
cp a.txt b.txt        # copy
mv a.txt archive/     # move or rename
rm notes.txt          # remove a file
rm -r projects        # remove a directory and its contents

Careful: rm does not use a recycle bin. Deleted means gone.

Summary

  • One tree, rooted at / — no drive letters.
  • pwd shows where you are; cd moves you.
  • Absolute paths start at /; relative paths do not.
  • ls, mkdir, cp, mv, rm manage files.
